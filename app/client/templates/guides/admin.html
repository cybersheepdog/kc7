Admin Guide

Full control
As an Admin you control the entire event end to end. This guide is organized by the order you'll usually do things: build a scenario → connect Azure → run the game → manage challenges & scoring → people & monitoring → outputs & resilience. Everything below lives in the Admin Central menu in the sidebar.
1 · Build a scenario
  • Manage Scenario — edit the actors, malware, company profile, and realism content pack in the browser. Every save runs the config validator first, so a bad config is rejected with clear errors instead of breaking generation.
  • Scenario Wizard — scaffold a validator-clean actor (and malware) from an archetype in a few clicks, then one-click its challenges and guide.
  • Scenario Templates — save the whole current scenario (company, actors, malware, content pack, challenges) as a reusable bundle, and load it later to spin the same event back up. Loading writes configs and adds challenges but never touches live scores.
2 · Connect Azure Data Explorer
  • ADX Configuration — enter your cluster URI, ingest URI, database, tenant, client ID and secret. Stored in the DB and applied immediately (no restart). GUI settings take priority over config.py.
  • Manage ADX Perms — grant players viewer access so they can run KQL in the Azure Data Explorer web UI.

No Azure cluster yet? In development, ADX_DEBUG_MODE prints generated data instead of uploading, so you can exercise the pipeline offline.

3 · Run the game
  • Manage the Game — start / stop / restart data generation with a live progress bar and a streamed log; you can cancel a long run mid-flight. Restart clears scores, solves, and generated data.
  • Schedule Game — for unattended events, auto-start generation and/or auto-stop scoring at set times (requires GAME_SCHEDULER_ENABLED).
  • Run History — review each run's timing, status, and per-table row counts.
4 · Challenges
  • Manage Challenges — create challenges by hand, bulk-import via CSV, or Auto-generate a set straight from the scenario's ground truth (malicious IPs/domains, phishing senders, malware families/hashes, attribution, ATT&CK ids). Run auto-generate after generating a game, since most facts only exist once data is produced.
  • Hints & gating — optionally give a challenge a point-cost hint, a timed unlock, and/or a prerequisite challenge. Anything you don't configure behaves exactly as before.
5 · Scoring & integrity
  • Score Audit — reconcile every player/team total against the recorded solves, indicator awards, hint costs and adjustments; ?apply=1 rebuilds scores and times from the records.
  • Manual Score Adjustments — add/subtract points with a reason (audited, reconcilable).
  • Re-grade — after fixing a challenge's accepted answers, retroactively credit players who answered correctly earlier (scored at their original time).
  • Answer Tester — confirm a challenge grades the way you expect before publishing, with a normalization breakdown.

These four grading tools are also available to the Grader role, so you can delegate adjudication.

6 · People & access
  • Manage Users — create accounts, reset passwords, change teams, and set each user's role: Player, read-only Observer, Grader (Observer + grading), or Admin. Roles are exclusive and never auto-assigned.
  • Manage Teams — create and manage teams and their aggregate scores.
  • Badges — most badges are earned automatically; here you grant or revoke the discretionary ones (MVP, Good sportsmanship, Team player). Audited.
  • Malicious Indicators — seed indicators for scoring when running against a pre-existing ADX dataset.

Note: if the seeded admin account still uses the default password, the app forces a password change before anything else can be done.

7 · Monitor the event
  • Live Answer Feed — real-time submissions plus advisory integrity flags (shared answers, fast copies, burst solving).
  • Analytics — solve rates, difficulty calibration, engagement, and ADX ingestion health.
  • Audit Log — append-only record of every privileged action.
8 · Outputs & resilience
  • Scenario PDF — a player packet or instructor answer key, generated from the live game.
  • Game Guide & Instructor Key — a Markdown brief and answer key built from the current config.
  • Export Game — archive final standings, solve stats, and the solve log as JSON/CSV.
  • Backup & Restore — download a full snapshot (configs + DB), or restore one; a DB restore is staged and swapped in safely on the next restart.
Automation & configuration

For reproducible / CI runs, the headless CLI drives the same pipeline without the web server: python kc7_cli.py validate | preview | generate (use --no-azure for a full offline dry-run).

Optional behaviors (dynamic scoring, live-push scoreboard, campaign mode, infrastructure reuse, per-technique alerts, indicator penalties, real-intel safety toggles) are feature flags in config.py — all off by default. See the README's "Feature flags" table.
